在 Linux 运维中，“网口不见了”（如找不到 eth0）是高频问题，绝大多数情况并非硬件故障，而是命名规则变更或驱动/配置问题。以下是系统性分析与排查指南： 🔍 核心原因分类（按发生频率排序）1️⃣ 网络接口命名规则变更（最常见！） 现象：系统实际存在网卡，但名称变为 enp0s3、ens192、eno1 等（systemd ≥ 197 的“可预测命名”）。 原因：现代发行版（CentOS 7+/RHEL 7+、Ubuntu 15.10+ 等）默认启用此规则，避免驱动加载顺序导致名称漂移。 验证：12ip link show # 查看所有接口（含 DOWN 状态）ls /sys/class/net/ # 直接列出内核识别的接口 若看到非 eth* 名称，即属此情况。 2️⃣ 驱动未加载或加载失败 驱动缺失/未编译进内核、initramfs 未包含驱动（内核升级后常见）、驱动被黑名单禁用、硬件不兼容。 排查：123lspci -k | grep -A 3 -i ethernet # 查看网卡及绑定驱动dmesg | ...
